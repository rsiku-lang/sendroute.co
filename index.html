Security notice — data storage
Member data (names, emails, phone numbers) is stored in your browser's localStorage. This is secure for single-device use but is not a substitute for a server-side database. Do not use this site on shared or public computers while logged into admin. For production scale, Version 5 moves all data to a secure server.
Share Admin Access With Your Team
The admin panel uses a single shared password. To give a team member access, share the password with them directly. To revoke access, change the password in the site code and redeploy via GitHub.
Current limitation: This is a single-file static site — there is no server-side user management. For multi-user role-based access (e.g. read-only for partners, write access for your team), the next step is to add a simple backend (Netlify Functions or Supabase). This can be built as Version 5 once you are live and generating revenue.
